Section 01 Introduction

1.1 Purpose of This Manual

This manual provides step-by-step operational guidance for administrators and operations staff who manage the EVODINE Restaurant Administration Panel. It covers all six core setup modules: Menu Categories, Food Portions, Tables, Chargers, Food Items, and Addons. By following this guide, staff can confidently configure the system so that the restaurant's front-of-house ordering experience runs smoothly.

  • Unrestricted access to all modules and system settings
  • Can enable or disable the code-edit feature for each module
  • Manages user accounts and role assignments
  • Responsible for initial system configuration and ongoing maintenance
🛠️ Restaurant Admin
  • Full create, read, update, and delete access to all six setup modules
  • Manages the daily menu configuration, table setup, and surcharge rules
  • Can upload and update food item and addon images
  • Receives toast notifications confirming successful operations
📋 Menu Manager
  • Can create, edit, and delete Categories, Portions, Food Items, and Addons
  • Read-only access to Tables and Chargers
  • Cannot modify surcharge rules or table capacity settings
👁️ Operations Staff
  • Read-only access to all setup modules for reference purposes
  • Cannot add, edit, or delete any records
  • Typically includes front-of-house supervisors reviewing menu data
Section 03 Menu Categories

3.1 Overview

Navigation Path: Sidebar Menu → Menu Setup → Categories  |  Intended Users: Restaurant Admin, Menu Manager

Menu Categories are the top-level groupings that organise your food items on the customer-facing ordering screen. Every food item must belong to a category, and every category must be linked to an order placement type (such as Dine In or Takeaway). Setting up categories correctly is the first step before any food items can be created.

3.2 Adding a New Category

  1. 1
    Navigate to Menu Categories

    From the left sidebar, expand the Menu Setup group and click Categories.

  2. 2
    Enter the Category Name

    In the Description field, type the name of your category (e.g., "Starters", "Main Course", "Beverages"). This field is required.

  3. 3
    Select the Order Placement Type

    From the Order Place Type dropdown, choose whether this category applies to Dine In, Takeaway, or another service mode. This field is required.

  4. 4
    Review the Code (optional)

    The system auto-generates a unique code for the category. If the code field is unlocked by your administrator, you may type a custom code. Otherwise, leave it blank.

  5. 5
    Click Save

    The system saves the record and displays a success notification showing the new category code. The list below the form refreshes automatically.

Form Field Reference

FieldTypeRequiredDescription
CodeTextAutoSystem-generated unique identifier. Editable only if code modification is enabled by the administrator.
DescriptionTextRequiredThe display name for the category (e.g., "Main Course").
Order Place TypeDropdownRequiredLinks the category to a service mode such as Dine In or Takeaway.

3.3 Editing & Deleting Categories

Editing a Category

  1. 1
    Locate the Category

    Scroll through the categories list or use the Search box at the top of the list to filter by name or code.

  2. 2
    Click the Edit Button

    Click the edit icon next to the category. The form above the list populates with the existing values.

  3. 3
    Modify the Fields and Click Update

    Make your changes and click Update. A success notification confirms the change, and the list refreshes.

Deleting a Category

  1. 1
    Click the Delete Button

    Click the delete icon next to the category you wish to remove.

  2. 2
    Confirm the Action

    A confirmation dialog appears asking "Do you want to delete this category?". Click Yes to proceed or No to cancel.

Important Deleting a category that already has food items linked to it may prevent those items from appearing correctly on the ordering screen. Always reassign or delete the linked food items before removing a category.

Section 04 Food Portions

4.1 Overview

Navigation Path: Sidebar Menu → Menu Setup → Food Portions  |  Intended Users: Restaurant Admin, Menu Manager

Food Portions define the size variants that can be assigned to food items — for example, Small, Regular, Large, Half, or Full. Each food item must have at least one portion assigned to it with a corresponding price before it can be saved. Portions are created here first and then selected during food item setup.

Note The Display Name field controls the label shown to customers on the ordering screen (e.g., "Sm", "Lg"). The Description is the full internal name used by staff.

4.2 Adding a New Portion

  1. 1
    Navigate to Food Portions

    From the left sidebar, expand Menu Setup and click Food Portions.

  2. 2
    Enter the Portion Description

    In the Description field, enter the full internal name (e.g., "Large Portion"). This field is required.

  3. 3
    Enter the Display Name

    In the Display Name field, enter the short name shown to customers (e.g., "Lg"). This field is required.

  4. 4
    Review the Code (optional)

    The system assigns a code automatically. If code editing is enabled, you may enter a custom code.

  5. 5
    Click Save

    A success notification confirms the save and shows the new portion code. The list below refreshes.

Form Field Reference

FieldTypeRequiredDescription
CodeTextAutoUnique identifier. Auto-generated; editable only if permitted by the administrator.
DescriptionTextRequiredFull internal name for the portion size (e.g., "Large Portion").
Display NameTextRequiredShort label shown on the customer ordering screen (e.g., "Lg").

4.3 Editing & Deleting Portions

  1. 1
    Find the Portion

    Use the search box or scroll through the portions list to locate the record you want to modify.

  2. 2
    Click Edit or Delete

    Click the edit icon to load the record into the form, or click the delete icon and confirm the action in the dialog that appears.

  3. 3
    Save Changes

    After editing, click Update. Click Clear at any time to reset the form without saving.

Warning Deleting a portion that is already assigned to one or more food items may affect the pricing and availability of those items. Review all linked food items before deleting a portion.

Section 05 Tables Management

5.1 Overview

Navigation Path: Sidebar Menu → Dining Setup → Tables  |  Intended Users: Restaurant Admin

The Tables module allows administrators to register all physical dining tables in the restaurant. Each table entry includes a name and a seating capacity. This data is used by the ordering system to assign orders to specific tables and manage dine-in seating efficiently.

5.2 Adding a New Table

  1. 1
    Navigate to Tables

    From the sidebar, click Tables under the Dining Setup section.

  2. 2
    Enter the Table Name

    In the Table Name field, type a recognisable name for the table (e.g., "Table 01", "Window Booth", "VIP Room"). This field is required.

  3. 3
    Enter the Head Count

    In the Head Count field, enter the maximum number of guests this table can seat. This field is required and must be a number.

  4. 4
    Review the Code (optional)

    A code is automatically assigned. It may be edited if the administrator has enabled this setting.

  5. 5
    Click Save

    The system saves the table and displays the new table code in a success notification. The tables list refreshes immediately.

Form Field Reference

FieldTypeRequiredDescription
CodeTextAutoUnique table identifier. Auto-generated; editable only if the administrator has enabled code modification.
Table NameTextRequiredA recognisable name for the table as it appears in the ordering system (e.g., "Table 05").
Head CountNumberRequiredThe maximum number of guests the table can accommodate.

  1. 1

Section 06 Chargers (Surcharges)

6.1 Overview

Navigation Path: Sidebar Menu → Billing Setup → Chargers  |  Intended Users: Restaurant Admin, Super Admin

Chargers are additional fees applied to customer bills — such as a service charge, delivery fee, or a government-mandated levy. Each charger can be defined as either a fixed amount or a percentage of the bill total. These are configured here and are applied automatically when orders are processed.

6.2 Charge Types Explained

💰 Fixed Amount A flat fee added to every bill, regardless of the order total (e.g., $2.00 delivery fee)
📊 Percentage A percentage of the subtotal applied to each bill (e.g., 10% service charge; value must be between 0 and 100)
Charge TypeSelectionValue AllowedExample
Fixed AmountAmountAny positive number2.50 → adds $2.50 to every bill
PercentagePercentage0 to 100 (inclusive)10 → adds 10% of the bill subtotal

6.3 Adding a New Charger

  1. 1
    Navigate to Chargers

    From the sidebar, click Chargers under the Billing Setup section.

  2. 2
    Enter the Charger Name

    In the Name field, enter a short reference name (e.g., "SVC"). This field is required.

  3. 3
    Enter the Description

    In the Description field, provide a clear explanation (e.g., "10% Service Charge on all dine-in orders"). Required.

  4. 4
    Select the Charge Type

    Choose Amount for a flat fee, or Percentage for a proportional charge. The amount/percentage field below resets when the type changes.

  5. 5
    Enter the Amount or Percentage Value

    Type the value. For Percentage type, the system enforces a maximum of 100. Required.

  6. 6
    Click Save

    The charger is saved and the list refreshes. A success notification displays the new charger code.

Form Field Reference

FieldTypeRequiredDescription
CodeTextAutoUnique identifier, system-generated. Editable only if enabled by the administrator.
NameTextRequiredShort reference name for the charge (e.g., "SVC", "DEL").
DescriptionTextRequiredFull description of the charge as it may appear on receipts.
Charge TypeDropdownRequiredSelect Amount (fixed flat fee) or Percentage (proportional).
Amount / PercentageNumberRequiredThe fee value. Must be 0–100 when Percentage is selected.
Warning When you change the Charge Type between Amount and Percentage, the value field is cleared automatically. Always re-enter the correct value after switching types.

6.4 Editing & Deleting Chargers

  1. 1
    Locate the Charger

    Use the search box to filter the list by name, description, or code.

  2. 2
    Click Edit

    The charger details load into the form. Modify the required fields and click Update.

  3. 3
    To Delete

    Click the delete icon. Confirm in the dialog that appears. The charger is removed and the list refreshes.

Section 07 Food Items

7.1 Overview

Navigation Path: Sidebar Menu → Menu Setup → Food Items  |  Intended Users: Restaurant Admin, Menu Manager

The Food Items module is the core of the menu setup. Each food item represents a dish or drink on the menu. A food item must be linked to a category and must have at least one portion assigned with a unit price before it can be saved. Optional features include an image upload and a Combo flag for set meals.

The screen uses a two-tab layout:

TabPurpose
Food Items ListDisplays all existing food items in a searchable data grid. Automatically shown when items exist.
Add / Edit Food ItemContains the entry form for creating a new item or editing an existing one. Automatically shown when no items exist yet.

7.2 Adding a New Food Item

  1. 1
    Navigate to Food Items

    From the sidebar, click Food Items. Switch to the Add Food Item tab if the list is shown.

  2. 2
    Enter the Item Name

    In the Name field, type the dish name exactly as it should appear on the menu (e.g., "Grilled Chicken Burger"). Required.

  3. 3
    Enter the Description

    In the Description field, provide a customer-facing description of the item. Required.

  4. 4
    Select a Category

    From the Category dropdown, choose the menu category this item belongs to. Required.

  5. 5
    Mark as Combo (optional)

    If this item is a set meal or combo offer, tick the Is Combo checkbox.

  6. 6
    Assign at Least One Portion with a Price

    See Section 7.3 for full instructions on portion assignment. This step is mandatory — the system will reject the form if no portions are assigned.

  7. 7
    Upload an Image (optional)

    See Section 7.4 for image upload instructions.

  8. 8
    Click Save

    The system saves the food item along with the assigned portions and image. A success notification confirms the save with the new item code.

Form Field Reference

FieldTypeRequiredDescription
CodeTextAutoSystem-generated item code. Editable only if code modification is enabled.
NameTextRequiredThe menu-facing name of the food item.
DescriptionTextRequiredA brief customer-friendly description of the dish.
CategoryDropdownRequiredThe menu category this item belongs to.
Is ComboCheckboxOptionalTick to mark this item as a combo or set meal.
PortionsSelection PanelRequiredAt least one portion must be assigned with a unit price.
ImageFile UploadOptionalA photo of the dish. Image files only (JPG, PNG, etc.).

7.3 Assigning Portions & Prices

Portions are managed using a selection panel within the food item form. The left side shows available portions; the right side shows the portions assigned to this item with their prices.

  1. 1
    Find a Portion

    In the portions panel, scroll through the available portions list or type in the Search Portions box to filter by name.

  2. 2
    Click to Select a Portion

    Click on a portion that is not yet added (unselected). A pricing popup appears.

  3. 3
    Enter the Unit Price

    In the popup, enter the price for this portion of the food item (e.g., 4.50 for a Small portion). The price must be greater than zero.

  4. 4
    Click Add

    The portion is added to the assigned list on the right with the price you entered. The portion is now marked as selected and cannot be re-added.

  5. 5
    Repeat for Additional Portions

    Add as many portions as required. Each portion can only appear once per food item.

  6. 6
    Remove a Portion (if needed)

    Click the remove button next to a portion in the assigned list. Confirm in the dialog. The portion returns to the available list.

Important You must assign at least one portion with a price greater than zero. Clicking Save without any portions assigned will show an error: "Select one of portion with price" and the item will not be saved.

7.4 Uploading a Food Image

  1. 1
    Click the Image Upload Area

    The image section displays a placeholder image. Click the upload button or the image area to open your file browser.

  2. 2
    Select an Image File

    Choose any image file (JPG, PNG, GIF, etc.). Only image file types are accepted; non-image files are automatically rejected with an error message.

  3. 3
    Review the Preview

    The selected image appears in the preview area immediately so you can verify it looks correct before saving.

  4. 4
    Clear the Image (optional)

Note When editing an existing food item, the system automatically loads and displays the previously saved image. You can replace it by uploading a new file, or leave it unchanged.

7.5 Editing & Deleting Food Items

Editing a Food Item

  1. 1
    Go to the Food Items List Tab

    All existing food items are displayed in the searchable data grid. Use the search box to filter by name, code, or category.

  2. 2
    Click Edit

    Click the edit icon on the item's row. The form switches to the Add/Edit tab, pre-populated with the item's existing data, image, and portions.

  3. 3
    Make Changes and Click Update

    Modify any fields, portions, or image as needed. Click Update when done. A success message confirms the update.

Deleting a Food Item

A food item can be deleted from two places:

  • From the Food Items List tab — click the delete icon on the item's row
  • From the Edit form — click the Delete button at the bottom of the form while editing

Both actions show a confirmation dialog before proceeding. Confirm to delete; cancel to return without changes.

Section 08 Addons

8.1 Overview

Navigation Path: Sidebar Menu → Menu Setup → Addons  |  Intended Users: Restaurant Admin, Menu Manager

Addons are optional extras that customers can add to their orders — such as extra sauces, toppings, side dishes, or premium ingredients. Each addon has a fixed unit price and is linked to a menu category. Unlike food items, addons do not use portions; they are sold as individual units at a single price.

Like Food Items, the Addons screen uses a two-tab layout:

TabPurpose
Addons ListDisplays all existing addons in a searchable grid. Shown automatically when addons exist.
Add / Edit AddonEntry form for creating or editing an addon. Shown automatically when no addons exist yet.

8.2 Adding a New Addon

  2. 2

  3. 3
    Enter the Description

  4. 4
    Enter the Unit Price

    In the Unit Price field, enter the price for one unit of this addon (e.g., 0.50). Required.

  5. 5
    Select a Category

  6. 6

  7. 7

For Restaurant Administrators

Frequently Asked Questions

Q: Can I have the same portion (e.g., "Large") assigned to multiple food items at different prices? Yes. Each food item maintains its own portion-price relationship. You can assign the "Large" portion to a burger at $8.50 and to a pizza at $12.00 — each price is stored independently for that food item.
Q: What happens to orders if I delete a food item that has been ordered before? Historical order records are preserved regardless of whether the food item is deleted from the setup. The item will no longer appear on the live menu for new orders, but past orders are not affected.
Q: Can I upload multiple images per food item? No. Each food item and addon supports a single image. To change the image, upload a new file while editing the item — the new image replaces the previous one.
Q: Is there a file size limit for image uploads? Yes. The maximum upload size is configured by the system administrator in the application settings. If an upload fails silently, contact IT to check the configured file size limit.
Q: Can I rename a category after food items have already been linked to it? Yes. Editing a category's description only changes the display name. All existing food items linked to that category remain linked and are not affected by the name change.
Q: Why does the system show "Successfully inserted. Code is ..." after saving? This confirmation message is shown to inform you of the automatically generated unique code assigned to your new record. This code is useful for referencing the record in reports or when contacting support.
