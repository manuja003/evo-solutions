Section 01
Introduction

1.1 Purpose of This Manual

This User Manual provides complete operational guidance for the Tag Evo Restaurant Order Management System — a modern, real-time digital platform that connects the front-of-house (waiters, cashiers) with the back-of-house (kitchen, bar) to streamline restaurant operations end to end.

The system eliminates paper-based order taking, reduces communication errors between staff, accelerates order fulfilment, and provides management with live operational visibility and financial analytics.

1.2 System Overview

  • ►Digital order creation by servers and waiters directly at tables or counters
  • ►Real-time kitchen and bar displays that update instantly as orders arrive
  • ►Live order movement tracking with status progression from queue through to serving
  • ►Integrated cashier and payment processing with multi-method payment support
  • ►Management dashboards with KPIs, revenue charts, and operational analytics
  • ►Structured cancellation workflows with approval and full audit trails

2.2 Detailed Role Descriptions

🛡 Administrator

The Administrator has unrestricted access to every module within the system. Intended for restaurant managers, owners, or system operators.

  • View and manage all orders regardless of table, server, or department
  • Monitor the complete Order Movement dashboard — sees all kitchen and bar orders simultaneously
  • Access the full Dashboard with financial KPIs, revenue trends, and analytics
  • View and action all cancellation approval requests
  • Access cashier and payment modules
👑 Kitchen Head

The Kitchen Head oversees all food preparation activities across the entire kitchen department.

  • View all active kitchen orders assigned to any chef in the kitchen
  • Monitor preparation status across all kitchen staff at once
  • Identify delayed or stuck orders across the entire kitchen
  • Cannot view bar or beverage orders
🍳 Chef (Kitchen Chef)

Kitchen Chefs see only the orders that have been assigned to them personally.

  • View orders assigned specifically to their user account
  • Update food item statuses through each stage: Start Preparing → Ready → Serve
  • Cannot see orders assigned to other chefs
  • Cannot view bar or beverage orders
🎊 Bar Head

The Bar Head oversees all beverage preparation and bar service activities.

  • View all active bar orders assigned to any bar chef across the station
  • Monitor beverage preparation status across all bar staff
  • Identify overdue bar orders and act on bottlenecks
  • Cannot view kitchen food orders
🍸 Bar Chef (Bartender)
  • View bar orders assigned specifically to their bar station
  • Update beverage item statuses through the preparation flow
  • Cannot see orders assigned to other bar staff
  • Cannot view kitchen food orders
📋 Server / Waiter
  • Create new orders for tables via the Stue Order module
  • Add food items, portions, add-ons, and customer details
  • Submit cancellation requests for active orders if needed
  • Cannot access kitchen or bar displays
  • Cannot process payments (unless granted dual access by management)
💳 Cashier
  • View the queue of orders ready for payment settlement
  • Review order items and apply applicable charges (service charge, tax, etc.)
  • Print bills and invoices for customers
  • Process payments via cash, card, or split payment methods
  • Close orders after successful payment settlement

Order Creation — Stue Order

Access: Menu → Order  |  Intended Users: Server / Waiter, Administrator

The Stue Order module is used by servers and waiters to create customer orders. It is the entry point of the order lifecycle — once an order is submitted here, it immediately flows to the kitchen and/or bar for preparation.

4.1 Creating a New Order — Step by Step

1
Select Your Server Identity
Upon entering the module, select yourself from the server list. This links the order to your account for tracking and accountability.
2
Select Order Type
Choose the service type: Dine In, Takeaway, Delivery, or Package.
3
Select Table (Dine-In only)
Use the Table Selector to choose an available table and specify the number of guests. Only tables showing as Available can be selected.
4
Link Customer (Optional)
Use the Customer Search panel to find and link an existing customer profile for loyalty tracking. This step can be skipped.
5
Add Food Items
Browse by category tab → select food item → choose portion size → set quantity → select any add-ons. Repeat for all ordered items.
6
Add Special Notes (Optional)
For items needing special preparation (e.g., "no spice," "well done"), use the item-level notes field.
7
Review & Submit Order
Verify all items, quantities, portions, and add-ons. Confirm the correct table and pax count. Click Submit Order. The system returns an Order Number on success.
8
Payment Decision
Choose Pay Now for immediate payment (takeaway / packages) — or Pay Later for the cashier to handle payment after the meal is served.

Section 05
Order Movement — Kitchen & Bar Tracking

Access: Menu → Order Movement  |  Intended Users: Chef, Kitchen Head, Bar Chef, Bar Head, Administrator

The Order Movement module is the heart of kitchen and bar operations. It provides a real-time live display of all active orders and their food items, showing exactly where each item stands in the preparation lifecycle. Updates happen instantaneously without any page refresh, using the system's real-time SignalR connection.

5.1 What Happens Immediately After an Order Is Placed

  • 1.The system saves the order to the database with all food items and quantities
  • 2.A real-time notification is broadcast via SignalR to all connected Order Movement screens within one second
  • 3.The new order immediately appears on the display with a highlight animation — no manual refresh needed
  • 4.Each food item starts in the Checking stage — the initial acknowledgment step
  • 5.Kitchen staff see kitchen items; bar staff see bar items — routing is automatic

5.2 How Orders Are Routed — Kitchen vs. Bar

The system automatically separates food items by department based on the destination assigned to each menu item. A single customer order may contain both kitchen food items and bar beverage items — these are split and routed simultaneously:

DestinationWho Sees ItExamples
KitchenChef, Kitchen Head, AdministratorGrilled Chicken, Pasta, Salads, Soups
BarBar Chef, Bar Head, AdministratorFresh Lime Juice, Cocktails, Mocktails, Coffee
ⓘ Dual-Department Orders The kitchen never sees bar items, and the bar never sees kitchen items. Each department only sees what is relevant to them. The Administrator is the only role with combined visibility of both departments.

5.4 The Complete Order Lifecycle — Preparation Flow

Every food and beverage item progresses through six clearly defined stages from order receipt to billing. The diagram below shows each stage, the action required, and who is responsible.

🔎 Checking Order received at station. Chef reviews. Kitchen / Bar
🕐 Queued Chef has taken the order. Waiting to start. Kitchen / Bar
🔥 Preparing Active cooking or mixing in progress. Kitchen / Bar
🆎 Ready to Serve Prepared & plated. Waiting for server to collect. Server / Waiter
🍽 Served Delivered to customer table. Server confirms. Server / Waiter
🧾 Billing All items served. Moves to cashier queue automatically. Cashier

Detailed Stage Descriptions:

1
🔎 Checking — Order Arrives at Station
The very first stage. The order has just been submitted by the server and appears on the kitchen or bar screen. All staff in the relevant department can see it. No preparation has started yet — this is the review and acknowledgment step.
Action By: Kitchen / Bar Staff
▶ Button shown to staff: "Take" — to accept the order into their personal queue
2
🕐 Queued — Chef Has Accepted the Order
A specific chef or bar chef has claimed the order. It is now linked to their personal account and removed from the general unassigned queue. Other staff no longer see it in their view. Preparation is next in line but has not yet started.
Action By: Assigned Chef / Bar Chef
▶ Button shown: "Start" — to begin active preparation
3
🔥 Preparing — Active Preparation in Progress
Cooking or beverage preparation is actively underway. This is the most time-sensitive stage — Kitchen Head and Bar Head monitor how long items stay here to identify delays. The duration in this stage directly affects customer wait time.
Action By: Assigned Chef / Bar Chef
▶ Button shown: "Ready" — when the item is fully prepared and plated
4
🆎 Ready to Serve — Awaiting Collection by Server
The food or drink is fully prepared and waiting at the pass. The responsibility now shifts from the kitchen/bar to the server. The server must collect the item and deliver it to the customer's table. Items should not remain here for more than 2–3 minutes.
Action By: Server / Waiter
▶ Button shown: "Serve" — after physically delivering to the customer
5
🍽 Served — Delivered to Customer Table
The item has been physically delivered to the customer's table and the server has confirmed this in the system. When every single item across both kitchen and bar in the order reaches this stage, the system automatically triggers the move to the billing stage.
Action By: System (auto-advances to Billing when all items are served)
6
🧾 Billing — Order Moves to Cashier Queue
The complete order has been fully served. The system automatically moves it to the Cashier module's billing queue. No further action is required from kitchen or bar staff. The cashier takes over from this point to process payment.
Action By: Cashier
✓ Cashier Queue Trigger Rule An order appears in the Cashier queue only when every food and beverage item in that order has reached the Served stage. If even one item is still being prepared or is ready but not yet served, the order will not appear for the cashier.

5.6 Pinned Orders

Up to 3 orders can be pinned to the left sidebar for priority tracking. Pinned orders stay visible at the top regardless of scroll position. Click the pin icon on any order card to pin it. Use "Unpin All" to clear all pins at once.

5.7 Selecting Multiple Items for Bulk Action

Check the checkbox next to individual food items, or click "Select All" to select every item in an order. Then click the bulk action button to advance all selected items to the next stage simultaneously — ideal when a chef finishes multiple dishes at the same time.

5.8 Table Change

If a customer moves to a different table: click "Change Table" on the order card → select the new table from the popup → confirm. The order is updated system-wide immediately.

5.9 Add More Items to an Existing Order

Click "Add More" on the order card → the food selector opens → select additional items and quantities → submit. New items appear with Checking stage and proceed through the normal preparation flow. A real-time notification instantly updates all connected screens.

5.10 Food Item Cancellation

Click the cancel icon next to a food item → enter a mandatory remark explaining the reason → confirm. The item is removed from the active preparation queue immediately.

⚠ Note Cancelling a food item from Order Movement removes only that specific item. To cancel an entire order, use the Order Cancellation Request module (Section 8).

7.5 Payment Validation Rules

Validation RuleCondition
Amount CoverageTotal paid amount must be greater than or equal to the Grand Total
Card Type RequiredA card type (Credit or Debit) must be selected for any card payment

ⓘ Submit Button State The Submit button remains grayed out and disabled until all validations pass. A red balance display indicates an insufficient payment amount — increase the entered amount to match or exceed the grand total.

7.6 Completing Payment

After clicking Submit, the system records the transaction with cashier identity and timestamp, removes the order from the billing queue, and marks the order as paid. The cashier may then click "Close Order" to formally close it in the system.

7.7 Early Pay Orders

Orders created with the "Pay Now" option (e.g., takeaways) are flagged as Early Pay. These orders can be processed for payment immediately after order creation — they bypass the normal "wait for all items to be served" rule and show a Close Order button after payment is completed.

10.4 Detailed Stage Explanations

🔎 Checking — Order Arrives at Station

The very first stage for every food and beverage item when an order is placed. The kitchen or bar screen highlights the new order arrival. All staff in the relevant department can see it in this stage. No preparation has started — this is the review and acknowledgment step before a chef takes ownership.

🕐 Queued — Accepted by Chef

A specific chef or bar chef has claimed the order by clicking "Take." The item is now linked to their user account and no longer visible in other chefs' unassigned queues. Preparation is up next — the item is waiting in the chef's personal queue.

🔥 Preparing — Active Preparation

Cooking or beverage mixing is actively in progress. This is the most time-sensitive stage. Kitchen Head and Bar Head actively monitor how long items remain here. Extended duration in this stage signals a potential bottleneck or delay that needs attention.

🆎 Ready to Serve — Awaiting Collection

The food or drink is fully prepared and plated at the pass. The kitchen's job is done — it is now the server's responsibility to collect and deliver. Items should not remain in this stage for more than 2–3 minutes to ensure food quality and customer satisfaction.

🍽 Served — Delivered to Customer

The item has been physically delivered to the customer's table and the server has marked it as served in the system. When every single item in the entire order reaches this stage across both kitchen and bar, the system automatically triggers the move to Billing.

🧾 Billing — Moved to Cashier Queue

The order is complete and has been automatically moved to the Cashier module's queue. No further action is needed from kitchen or bar staff. The cashier takes over from here to review the order, apply charges, print the bill, and process payment.

Section 11
Best Practices

11.1 Order Creation — Server Best Practices

PracticeRecommendation
Double-check before submittingAlways review the full order — items, quantities, portions, and add-ons — before clicking Submit. Corrections after submission are more disruptive to kitchen workflow.
Use notes for special requestsAlways add preparation notes for dietary requirements or special requests. Do not rely on verbal communication with the kitchen.
Confirm guest countAlways enter the correct number of guests. This helps kitchen planning, portion preparation, and table management analytics.
Link customers when possibleLinking orders to customer accounts enables loyalty tracking and helps management identify repeat customers on the dashboard.
Use "Add More" — not new ordersIf a customer wants additional items, always use the Add More function on the existing order. Never create a second separate order for the same table.

11.2 Kitchen & Bar Operations Best Practices

PracticeRecommendation
Take orders immediatelyAs soon as a new order appears in Checking stage, take it into your queue without delay. This prevents confusion over unclaimed orders during busy periods.
Update statuses in real-timeAdvance stages promptly when you act. Delays in status updates create inaccurate data for managers monitoring wait times and for servers waiting to collect orders.
Use bulk selectionWhen multiple items for the same order are ready simultaneously, use "Select All" and advance them together rather than updating one at a time.
Pin priority ordersPin VIP table orders or time-sensitive orders using the pin feature to keep them visible at the top of your screen at all times.

11.3 Cashier Best Practices

PracticeRecommendation
Verify items before printingAlways open the order details and review all items and quantities before printing the customer's bill — avoid printing errors.
Apply charges consistentlyApply the same standard charges (e.g., service charge) to all dine-in orders unless management instructs otherwise for specific situations.
Use "Fill Exact Amount" for cardsFor card-only transactions, use the Fill Exact Amount button to auto-fill the correct amount and avoid manual entry errors.
Verify split payment totalsBefore submitting split payments, always confirm the balance display shows zero or positive — this confirms total coverage.

12.6 Frequently Asked Questions

Q: Can a server see the kitchen or bar display? No. Servers do not have access to the Order Movement module. This is by design to maintain a clear separation of responsibilities between front-of-house and back-of-house staff.
Q: What happens if the browser is closed while there is an active order? Active orders are not affected by closing the browser. All order data is stored on the server. Simply re-open the browser, navigate back to the module, and all current orders will be visible exactly as before.
Q: Can a cashier process payment before all items have been served? Under normal workflow, no — an order only appears in the cashier queue when every item is confirmed as served. The exception is Early Pay orders (such as takeaways created with "Pay Now"), which can be settled immediately after order creation.
Q: Can multiple cashiers be logged in and working at the same time? Yes. Multiple cashiers can be logged in simultaneously and the billing queue updates in real-time for all of them. It is recommended to assign specific cashiers to specific orders to prevent two cashiers from processing the same bill.
Q: Will the kitchen see add-more items immediately? Yes. When a server adds more items to an existing order, the kitchen and bar displays update within seconds through the real-time connection — no manual refresh is required.
Q: Can an approved cancellation be reversed? No. Once a cancellation request is approved, the order cancellation is final. If the order needs to be re-entered, a completely new order must be created from the Stue Order module.
Q: The screen is not showing new orders automatically — what should I do? The real-time connection may have dropped temporarily. Refresh the browser page — the system will automatically reconnect and reload all current order data within a few seconds.
